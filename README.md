# RandomPath · 随机骑行路线生成器

小沫陪哥哥一起探索骑行路线 ✨🚴

一个基于 Vue 3 的 PWA 骑行路线规划工具，支持随机通勤路线生成、环形骑行路线生成，以及 90+ 条预置经典骑行路线。使用高德地图 API 进行路线规划，Open-Meteo 获取高程数据。

---

## 项目概览

| 项目 | 说明 |
|------|------|
| 技术栈 | Vue 3 + Vue Router 4 + Vite 6 |
| PWA 支持 | vite-plugin-pwa，支持离线缓存 |
| 地图 API | 高德地图 Web API v3/v5（骑行路线规划、地理编码、POI 搜索） |
| 高程 API | Open-Meteo Elevation API |
| 部署 | Vercel（支持自定义 base 路径） |
| 路由模式 | Hash 模式（`#/commute`、`#/loop`、`#/preset`） |

---

## 路由结构

| 路由 | 页面 | 功能 |
|------|------|------|
| `#/commute` | CommuteView | 🎲 随机通勤路线生成 |

## 功能点详解

### 一、通用功能（三个页面共享）

#### 1. Toast 消息提示系统
- **位置**：`App.vue` 中全局注册 `window.$toast(msg, type)`
- **类型**：普通消息（粉色）、警告（橙色 `warn`）、错误（红色 `err`）
- **行为**：从顶部滑入，2.2 秒后自动消失
- **调用方式**：各组件内 `const toast = (m, t) => window.$toast?.(m, t)`

#### 2. 地址簿系统（LocalStorage 持久化）
- **存储键**：`radompath_addresses_v4`
- **数据结构**：`{ 别名: { name, lng, lat } }`
- **功能**：
  - 添加新地址（别名 + 名称 + 经纬度）
  - 地址名称可通过高德地理编码自动查询坐标
  - 坐标可手动填写或通过 [高德坐标拾取器](https://lbs.amap.com/tools/picker) 获取
  - 在通勤/环线/经典路线页面均可快速调用已保存的地址
- **初始状态**：无默认地址，需用户自行添加

#### 3. 地址输入自动补全
- **实现**：`useAutoComplete.js`，基于高德 `AMap.AutoComplete` 插件
- **行为**：
  - 输入地名时实时搜索，限西安市范围（`citylimit: true`）
  - 下拉列表显示地名 + 所属区域
  - 选中后自动填入名称和经纬度
  - 失焦 200ms 后关闭下拉

#### 4. 地理编码（地名 ↔ 坐标）
- **正向编码** `geocode(address, city)`：地名 → 经纬度，默认搜索西安
  - 带缓存机制（Map，最多 100 条）
- **反向编码** `reverseGeocode(lng, lat)`：经纬度 → 地名
- **API**：高德 `v3/geocode/geo` 和 `v3/geocode/regeo`

#### 5. GPS 定位功能
- 调用浏览器 `navigator.geolocation` API
- 高精度模式（`enableHighAccuracy: true`），10 秒超时
- 获取坐标后自动反向编码为地名
- 失败时提示检查权限

#### 6. 路线缩略图 Canvas 绘制
- **组件**：`RouteThumbnail.vue`
- **绘制内容**：
  - 路线轨迹（粉色半透明线条 + 实线描边）
  - 起点（绿色圆点 + "起" 字标注）
  - 终点（橙色圆点 + "终" 字标注）
  - 途经点（青色圆点 + "途经N" 标注）
  - 补给点（紫色圆点 + 序号标注）
  - 指北箭头（↖ 固定位置）
- **交互功能**：
  - 🟣 **缩放**：＋／－ 按钮，最大 20 倍放大，最小恢复 1:1
  - 🖐 **拖拽平移**：放大后可用鼠标/触屏拖拽画布（仅 zoom > 1 时生效）
  - 🔄 **重置**：一键恢复 1:1 视图
  - 🟣 **补给点点击**：点击紫色补给点，触发高亮闪烁（金环脉冲动画 x3），同时弹出 Toast 显示名称
  - 📜 **自动滚动**：点击补给点时画布自动滚动到可视区域

#### 7. 路线结果展示面板（通用结构）
- **统计卡片**：总距离（km）、预计时间（分钟）、难度等级
- **路线缩略图**（`RouteThumbnail`）
- **路线摘要**：起点 → 途经点1 → 途经点2 → ... → 终点
- **折叠详情**（▶ 详细数据，点击展开）：
  - 爬升高度（m）、卡路里消耗（kcal）、途经点数量
  - 分段明细：每一段的起终点名称 + 距离 + 时间
  - 途经点坐标列表
- **导航按钮**：跳转高德地图 APP 导航
- **导航链接**：可复制的高德导航 URL
- **复制按钮**：一键复制导航链接
- **GPX 导出**：下载 GPX 轨迹文件（可导入码表/Garmin 等设备）
- **换一条 / 重新生成按钮**

#### 8. 导航功能
- **生成高德导航 URL**（`buildNavUrl`）：基于高德地图 `ridemap` 协议，包含起点、终点、途经点坐标和名称
- **跳转导航**（`openNavigation`）：`window.open` 跳转到高德地图 APP
- **途经点支持**：最多支持多个途经点拼接在 URL 中

#### 9. GPX 轨迹导出
- **格式**：标准 GPX 1.1 XML，包含 `<trk>` 轨迹段
- **内容**：所有路段的 polyline 点序列
- **命名规则**：`RandomPath_{类型}_{起点}_{终点}_{距离}km.gpx`

#### 10. 卡路里计算
- **公式**：`8 × 体重(kg) × (骑行时长/3600)`
- **体重来源**：LocalStorage `radompath_weight`，默认 70kg
- **单位**：kcal

#### 11. 难度评级
- **评级标准**（综合距离 + 爬升比）：
  | 条件 | 评级 | 颜色 |
  |------|------|------|
  | ≥120km 或 爬升比 >15m/km | ★★★★★ 极限 | 红色 |
  | ≥80km 或 爬升比 >10m/km | ★★★★ 困难 | 橙色 |
  | ≥50km 或 爬升比 >6m/km | ★★★ 中等 | 黄色 |
  | ≥25km 或 爬升比 >3m/km | ★★ 进阶 | 绿色 |
  | 其他 | ★ 休闲 | 蓝色 |

#### 12. 途经点智能命名
- **`nameWaypoint(lng, lat)`**：三级回退策略
  1. 反向地理编码获取地址，过滤掉小区/家属院/学校等非地标名称
  2. 搜索周边 POI（道路交叉口、公园广场、风景名胜、地标、商场等），取第一个非住宅类 POI
  3. 再次反向地理编码，去除括号内小区名
  4. 兜底显示为坐标 `"经度, 纬度"`
- **住宅过滤关键词**：小区、家属院、花园、学校、公寓、停车场、车库 等
- **QPS 控制**：每步操作间有 200ms~600ms 延迟

#### 13. 全局 Toast & 加载进度环
- 生成路线时显示 SVG 圆环进度条（0→100%）
- 显示当前尝试次数和状态提示
- 10 个圆点表示每次尝试的结果：✅ 成功（绿）、❌ 失败/超出范围（红）、🔄 当前进行中

#### 14. PWA 离线支持
- Service Worker 缓存策略（`vite-plugin-pwa`）：
  - 高德 API：NetworkFirst，缓存 1 天，最多 50 条
  - Open-Meteo 高程 API：NetworkFirst，缓存 7 天，最多 200 条
- 可安装到手机主屏幕（`display: standalone`）

### 二、随机通勤页面（CommuteView — `#/commute`）

#### 1. 起终点输入
- **起点**：地名输入 + 自动补全 + 地址簿快速选择 + 地理编码按钮 + GPS 定位按钮
- **终点**：同上
- **经纬度**：手动输入框（支持 6 位小数精度）
- 自动填充：页面加载时从地址簿读取「家」→ 起点，「公司」→ 终点

#### 2. 扇区模式选择
控制随机途经点的偏移方向：

| 模式值 | 含义 |
|--------|------|
| `mixed`（混合） | 随机选择左偏或右偏，每次独立 |
| `0`（左侧·近） | 固定左侧偏移，较小偏移量 |
| `1`（左侧·远） | 固定左侧偏移，较大偏移量 |
| `2`（右侧·近） | 固定右侧偏移，较小偏移量 |
| `3`（右侧·远） | 固定右侧偏移，较大偏移量 |
| `random`（随机扇区） | 从 4 个扇区中随机，且避免与最近 2 次重复 |

- 扇区记忆：最近 5 次历史记录中的扇区用于避免重复

#### 3. 最大距离限制
- 用户可设定最大公里数（默认 50km）
- 生成时路线总距离不超过此值

#### 4. 随机途经点生成算法（`generateWaypoints`）
- **途经点数量**：1~10 个随机
- **偏移逻辑**：
  1. 计算起终点方位角（bearing）和直线距离
  2. 根据扇区模式决定偏移方向（左右 ±90°）和偏移量
  3. 每个途经点在直线方向上按比例分布（沿走廊方向排序）
  4. 偏移距离 = 起点到该点投影距离 × (1.5%~35%) 随机系数
- **目标距离**：在直线距离的 1.35 倍与用户设定最大值之间随机取
- **余量分配**：剩余距离均分给各途经点偏移

#### 5. 多段骑行路线并发查询
- 将途经点按顺序插入起终点之间，形成 N 段
- **并发请求**：所有段同时调用高德骑行路线 API（`Promise.all`）
- 每段返回：距离、时长、polyline（轨迹点序列）

#### 6. 智能重试机制（最多 30 次）
- **重试逻辑** `tryGenerateRoute`：
  - 每次生成不同途经点组合（随机种子不同）
  - 如果总距离在范围内 → 进入修复流程
  - 如果超出范围 → 记录但是继续尝试
- **早停阈值**：第 8 次（`EARLY_ACCEPT_AFTER`）起，任何可接受的结果都会被提前采用
- **最优策略**：记录距离目标最近的一次结果作为备选

#### 7. 折返/死胡同检测与修复
- **绕路比检测**（`detourRatio`）：每段实际距离 / 直线距离 > 2.5 判定为绕路
- **折返检测**（`polylineOverlap`）：相邻两段 polyline 尾部与头部在 300 米内有≥3 个重合点，判定为折返
- **死胡同修复**（`tryFixDeadEnds`）：最多 3 轮
  1. 找出所有满足"绕路比 > 2.0 或折返重叠"的途经点
  2. 将该途经点"吸附"到前后两段 polyline 的交汇处（距离 < 50m 的点之前）
  3. 重新查询修正后的路段
  4. 修复后若仍在范围内 → 立即接受
- **修复上限**：最多 3 轮，每轮最多修正所有死胡同途经点

#### 8. 高程爬升计算
- 合并所有路段的 polyline，采样最多 20 个点
- 调用 Open-Meteo Elevation API 获取每个采样点的高程
- 累加正高差作为总爬升（仅 `diff > 0` 的部分）
- 失败时返回 `null`，界面显示 `--`

#### 9. 路线历史记录
- **存储键**：`radompath_history`
- **记录内容**：路线类型、起终点名称、距离、扇区、途经点列表
- **最多保存**：20 条
- **用途**：扇区去重（避免连续生成相同扇区的路线）

### 三、环线骑行页面（LoopView — `#/loop`）

#### 1. 中心点输入
- 地名 + 自动补全 + 地址簿快速选择 + 地理编码 + GPS 定位
- 环线起终点相同（从中心点出发回到中心点）

#### 2. 目标距离选择
- **快捷选择**：15km / 30km / 50km / 80km / 100km 按钮组
- **自定义输入**：10~120km，步长 5km
- **默认值**：30km

#### 3. 环形途经点生成算法（`generateLoopWaypoints`）
- **途经点数量**：3~6 个随机
- **环半径计算**：
  - 首次生成：目标距离 / 7
  - 后续重试：在目标距离的 50% ~ 最大范围之间随机 / 8.2
- **分布方式**：均匀放射状分布 + 随机角度偏移（±12.5°）+ 随机半径抖动（75%~125%）
- **排序**：按方位角顺时针排序
- **允许范围**：目标距离的 60%~140%（容错带）

#### 4. 距离快捷切换
- 点击预设距离按钮立即切换目标距离
- 支持手动输入任意值

### 四、经典路线页面（PresetView — `#/preset`）

#### 1. 90+ 条预置路线库
- **数据**：`PRESET_ROUTES` 数组，每条路线包含名称、起点、终点、途经点坐标和名称
- **分类**：西安（约 50 条）、咸阳、关中、秦岭（约 13 条）、宝鸡、渭南、汉中、延安、铜川、商洛、安康

#### 2. 路线搜索 & 分组
- **搜索过滤**：输入关键词实时过滤（匹配路线名 / 起点名 / 途经点名）
- **分组显示**：下拉列表按城市分组，使用 `<optgroup>` 标签
- **分组顺序**：西安 → 咸阳 → 关中 → 秦岭 → 宝鸡 → 渭南 → 汉中 → 延安 → 铜川 → 商洛 → 安康

#### 3. 路线详情预览
- 选择路线后展示：起点名称+坐标、终点名称+坐标、途经点数量
- 途经点可视化链：起点 → 途经 → 途经 → ... → 终点（颜色编码标签）
- 当途经点超过 6 个时折叠显示 `... (N+ more)`

#### 4. 自定义起点
- 可选择地址簿中的地址替换默认起点
- 支持手动输入名称 + 经纬度
- 也支持自动补全搜索

#### 5. 途经点编辑
- 预置路线的途经点可查看、修改名称和坐标
- 支持新增自定义途经点
- 支持删除任意途经点（红色 X 按钮）

#### 6. 分段路线查询
- 所有点（自定义起点 → 途经点1 → 途经点2 → ... → 终点）串行分段查询
- 每段调用高德骑行路线 API，段间有 800ms 延迟（防 QPS 限流）
- 进度条显示当前查询到第几段

#### 7. 沿途补给点搜索
- **首次点击**「🔍 搜索沿途补给点」按钮：
  1. 从路线 polyline 中采样最多 9 个点
  2. 每次搜索 3 个采样点周围 800 米范围的 POI
  3. POI 类型：便利店/超市（`080100`）+ 餐饮（`010100`）+ 购物中心（`060000`）+ 公厕/服务区（`050000`）
  4. 自动去重（名称+四舍五入坐标）
- **再次点击**「🔄 加载更多补给点」：继续搜索下一批 3 个采样点
- **结果展示**：
  - 紫色标签列表，点击高亮 + 缩略图上闪烁
  - 缩略图上紫色圆点标注位置，带序号
- **QPS 控制**：每次搜索间隔 1 秒

#### 8. 缩略图补给点交互
- 点击紫色标签 → 缩略图上对应补给点金色闪烁（脉冲 3 次）
- 点击缩略图上补给点 → Toast 显示名称
- 自动重置缩放并滚动到可视区域

## 架构设计

### 文件结构
```
src/
├── main.js                          # Vue 入口 + 路由配置
├── App.vue                          # 根组件（Header、TabBar、Toast、Footer）
├── views/
│   ├── CommuteView.vue              # 随机通勤页面
│   ├── LoopView.vue                 # 环线骑行页面
│   └── PresetView.vue               # 经典路线页面
├── components/
│   └── RouteThumbnail.vue           # Canvas 路线缩略图组件
├── composables/
│   ├── useAMap.js                   # 高德地图 API 封装
│   ├── useAutoComplete.js           # 地址自动补全
│   ├── useRouteEngine.js            # 核心路线生成引擎
│   ├── useScoring.js                # 路线评分 & 难度评级
│   └── useStorage.js                # LocalStorage 读写封装
└── utils/
    └── math.js                      # 地理数学工具函数
```

### 数据流
```
用户输入（起终点/中心点/预设路线）
  → 高德地理编码（地名→坐标）
  → 随机途经点生成（扇区/环线算法）
  → 高德骑行路线 API（分段 polyline 查询）
  → 死胡同检测 & 修复
  → Open-Meteo 高程查询
  → 结果组装 & 展示
  → 导航/GPX/补给点搜索
```

### 关键配置常量
| 常量 | 值 | 说明 |
|------|-----|------|
| `MAX_WAYPOINTS` | 10 | 单次生成最多途经点 |
| `MAX_RETRIES` | 30 | 路线重试次数上限 |
| `EARLY_ACCEPT_AFTER` | 8 | 早停接受阈值 |
| 通勤默认最大距离 | 50km | |
| 环线默认目标距离 | 30km | |
| 环线距离容差 | 60%~140% | |
| 高德 API 缓存 | 1 天 / 50 条 | PWA Workbox |
| 高程 API 缓存 | 7 天 / 200 条 | PWA Workbox |
| 历史记录上限 | 20 条 | |
| 地址簿存储键 | `radompath_addresses_v4` | |

## 🐛 已知问题修复记录（2026-07-07）

### 问题 1：添加的地址无法删除，重新打开后丢失

**根因**：
- `useStorage.js` 没有 `deleteAddress` 函数
- `loadAddresses()` 在 localStorage 数据格式异常时直接覆盖为空对象 `{}`，导致已保存地址丢失

**修复**：
- `useStorage.js`：新增 `deleteAddress(alias)` 函数，`saveAddresses` 改为先转普通对象再存储，`loadAddresses` 增加数据类型校验（非对象或数组时仅重置不覆盖）
- `CommuteView.vue`：地址簿弹窗改为「管理地址簿」，顶部显示已有地址列表 + 🗑 删除按钮，底部保留添加功能

### 问题 2：添加地址定位不准，输入框提示不清晰

**根因**：
- `geocodeNewAddr()` 调用 `geocode(n)` 时**未传城市参数**（对比 `doGeocode` 传了 `'西安'`），导致全国范围匹配精度下降
- 地址弹窗 4 个字段无步骤引导，用户不知道先填哪个
- 起点/终点输入框 placeholder 只写「地名」，不够具体

**修复**：
- `geocodeNewAddr`：传入 `'西安'` 城市参数，坐标为 `String()` 类型确保与输入框兼容
- 地址弹窗 UI：增加步骤编号 ①②③ 标签，placeholder 加示例，底部加 💡 操作提示
- 起点/终点输入框：placeholder 改为「输入地名搜索（如：西安钟楼）」，经纬度 placeholder 改为「自动填入或手动输入」

### 问题 3：经典模式缩略图放大缩小还原拖拽，缩略图异常循环

**根因**：
- zoom 值无 NaN/Infinity 保护，浮点精度累积可能导致异常状态
- pan 值无上限，无限拖拽导致画布内容漂移出可视区域
- zoom 变换矩阵中心点与 pan 耦合，缩放时画面偏移方向不可预期
- `flashMarker` 内 `resetView()` 与 zoom/pan 的 `watch` 可能产生连锁重绘

**修复**：
- `RouteThumbnail.vue`：新增 `clampZoom()` / `clampPan()` 安全函数，zoom 范围 `[1, 20]`，pan 范围 `[-2000, 2000]`
- zoom 变换改为以 canvas 中心缩放 + 独立 pan 偏移，二者解耦
- 「－」按钮缩放逻辑：`zoom/2` → `clampZoom()`，pan 等比缩放，zoom=1 时自动归零 pan
- `highlightIndex` watch 增加 `flashTimer` 守卫，避免重复触发 `resetView`

### 问题 4：前两个模式生成的路径途经点老是拐进去又拐出来（折返）

**根因**：
- `generateWaypoints` 中偏移系数 `bc` 最大达 **0.35**（35% 直线距离），乘以随机系数 1.4 后单点偏移可达 49%
- 最大偏移距离 `od` 为 **8000m**，对于 20km 通勤意味着途经点可能偏离主路近 8 公里
- 折返检测阈值过松：绕路比 > 2.5 才判定折返，死胡同修复的绕路比阈值为 2.0

**修复**：
- `generateWaypoints`：`bc` 上限 0.35→**0.08**（偏移最多 8%），`od` 上限 8000→**2000m**，`cf` 随机系数 0.8→0.6
- `checkBacktrack`：绕路比阈值 2.5→**2.0**，折返重叠距离 300m→**200m**
- `findDeadEndWaypoints`：绕路比阈值 2.0→**1.6**，折返重叠距离 300m→**200m**
